# VLM 모델 리서치 중간 결과

## 요구사항 정리
1. JSON 출력 지원
2. 16k 이상의 컨텍스트
3. H100 80GB에서 학습 가능한 모델 크기
4. 다중 이미지 입력 지원

## 발견된 주요 정보

### JSON 구조화된 출력 지원
- **Outlines 라이브러리**: Pydantic 모델을 사용하여 VLM의 JSON 구조화된 출력 지원
- **SmolVLM-Instruct**: HuggingFace에서 구조화된 출력 예제로 사용됨
- **vLLM**: 구조화된 출력을 위한 guided decoding 지원 (outlines, lm-format-enforcer, xgrammar 백엔드)

### GPU 메모리 요구사항 (H100 80GB)
- 일반적인 가이드라인: 풀 파인튜닝은 파라미터 1B당 약 16GB VRAM 필요
- 7B 모델: 약 112GB (풀 파인튜닝), LoRA/QLoRA로 대폭 감소 가능
- H100 80GB: 7B 이하 모델을 LoRA/QLoRA로 파인튜닝 가능, 또는 3B 이하 모델 풀 파인튜닝 가능

### 다중 이미지 입력 지원
- **LEOPARD**: 텍스트가 풍부한 다중 이미지 작업을 위한 VLM
- **Qwen2-VL**: 다중 이미지 입력 지원
- **Pixtral**: 다중 이미지 지원
- **LLaVA**: 원본 모델은 다중 이미지에 약하지만, 일부 변형 모델은 지원
- **Idefics2**: 다중 이미지 입력 지원

### 긴 컨텍스트 지원
- **InternVL2-V2PE-32K**: 32K 컨텍스트 지원
- **Qwen2-VL**: 32K 토큰까지 확장 가능
- 대부분의 최신 VLM은 8K-32K 범위의 컨텍스트 지원

## 다음 단계

3. HuggingFace에서 실제 모델 페이지 방문하여 검증


## Qwen2.5-VL 상세 정보

### 모델 스펙
- **모델 크기**: 3B, 7B, 72B 파라미터 버전 제공
- **컨텍스트 길이**: 
  - 기본: 32,768 토큰
  - YaRN 기법 사용 시: 128K 토큰까지 확장 가능
  - 긴 비디오 입력: 64K 토큰까지 직접 수정 가능
- **다중 이미지 지원**: ✅ 다중 이미지 및 비디오 입력 지원
- **JSON 구조화 출력**: ✅ 구조화된 출력 지원 (송장, 양식, 표 등 스캔 데이터)
- **라이센스**: Apache 2.0

### 주요 기능
1. **시각적 이해**: 텍스트, 차트, 레이아웃 분석 가능
2. **긴 비디오 이해**: 1시간 이상의 비디오 이해 가능
3. **동적 해상도**: 임의의 이미지 해상도 처리 가능 (4-16384 비주얼 토큰)
4. **구조화된 출력**: JSON 형태의 안정적인 출력 생성

### 파인튜닝 지원
- ✅ 파인튜닝 가능
- Transformers 라이브러리 지원
- 다중 이미지 데이터셋으로 파인튜닝 가능

### GPU 메모리 요구사항
- **7B 모델**: 약 16GB (추론 시 bfloat16)
- **파인튜닝**: LoRA/QLoRA 사용 시 H100 80GB에서 충분히 가능
- **양자화**: int4, int8 양자화 지원으로 메모리 절감 가능

### 벤치마크 성능 (Qwen2.5-VL-7B)
- DocVQA: 95.7
- InfoVQ: 82.6
- ChartQA: 87.3
- TextVQA: 84.9
- OCRBench: 864
- MMStar: 63.9

### 출처
- 모델 카드: https://huggingface.co/Qwen/Qwen2.5-VL-7B-Instruct
- 문서: https://huggingface.co/docs/transformers/en/model_doc/qwen2_5_vl

## Pixtral-12B 상세 정보

### 모델 스펙
- **모델 크기**: 12B 파라미터 + 400M 비전 인코더
- **컨텍스트 길이**: 128K 토큰
- **다중 이미지 지원**: ✅ 가변 이미지 크기 지원
- **라이센스**: Apache 2.0

### 주요 기능
1. **네이티브 멀티모달**: 이미지와 텍스트가 인터리브된 데이터로 학습
2. **자연스러운 해상도**: 이미지를 자연스러운 해상도와 종횡비로 처리
3. **문서 이해**: 자연 이미지 및 문서 이해에 특화

### 제약사항
- ❌ 중재 메커니즘 없음
- 12B 파라미터로 H100 80GB에서 파인튜닝 가능하지만 7B보다 메모리 요구량 높음

### 출처
- 모델 카드: https://huggingface.co/mistralai/Pixtral-12B-2409

## InternVL2-8B 상세 정보

### 모델 스펙
- **모델 크기**: 8.1B 파라미터 (InternViT-300M-448px + internlm2_5-7b-chat)
- **컨텍스트 길이**: 8K 토큰
- **다중 이미지 지원**: ✅ 다중 이미지, 긴 텍스트, 비디오 입력 지원
- **라이센스**: MIT

### 주요 기능
1. **문서 및 차트 이해**: 문서, 차트, 인포그래픽 QA에 강점
2. **OCR 및 장면 텍스트 이해**: 텍스트 인식 및 이해 능력
3. **과학 및 수학 문제 해결**: 과학적, 수학적 문제 해결 능력
4. **다중 라운드 대화**: 이미지/비디오와 함께 다중 턴 대화 지원

### 파인튜닝 지원
- ✅ 파인튜닝 가능 (InternVL, SWIFT, XTurner 등 지원)
- 8비트 양자화 지원 (BNB)
- 멀티 GPU 추론 지원

### GPU 메모리 요구사항
- **8B 모델**: bfloat16으로 약 16GB
- **파인튜닝**: LoRA/QLoRA 사용 시 H100 80GB에서 충분히 가능

### 제약사항
- ❌ 컨텍스트 길이가 8K로 제한적 (16K 요구사항 미충족)

### 출처
- 모델 카드: https://huggingface.co/OpenGVLab/InternVL2-8B

## LLaVA-OneVision-Qwen2-7B 상세 정보

### 모델 스펙
- **모델 크기**: 0.5B, 7B, 72B 파라미터 버전 제공 (7B 버전: 8.03B params)
- **컨텍스트 길이**: 32K 토큰
- **다중 이미지 지원**: ✅ 단일 이미지, 다중 이미지, 비디오 입력 지원
- **라이센스**: Apache 2.0

### 아키텍처
- **비전 인코더**: SO400M
- **언어 모델**: Qwen2 (32K 컨텍스트)
- **정밀도**: bfloat16

### 학습 데이터
- **Pretraining**: LCS-558K (1 epoch, projector만)
- **Mid Stage**: 4.7M 고품질 합성 데이터 (1 epoch, 전체 모델)
- **Final-Image Stage**: 3.6M 단일 이미지 데이터 (1 epoch, 전체 모델)
- **OneVision Stage**: 1.6M 단일/다중 이미지/비디오 데이터 (1 epoch, 전체 모델)

### 주요 기능
1. **멀티모달 상호작용**: 이미지, 다중 이미지, 비디오와 상호작용 가능
2. **긴 컨텍스트**: 32K 토큰으로 긴 대화 및 복잡한 입력 처리
3. **다국어 지원**: 영어, 중국어

### 파인튜닝 지원
- ✅ 파인튜닝 가능
- Huggingface Trainer 사용
- 256 × Nvidia Tesla A100으로 학습됨

### GPU 메모리 요구사항
- **7B 모델**: bfloat16으로 약 16GB
- **파인튜닝**: LoRA/QLoRA 사용 시 H100 80GB에서 충분히 가능

### 출처
- 모델 카드: https://huggingface.co/lmms-lab/llava-onevision-qwen2-7b-ov
- GitHub: https://github.com/LLaVA-VL/LLaVA-NeXT

## 최종 추천 모델

### 1순위: Qwen2.5-VL-7B-Instruct
**선정 이유:**
- 모든 요구사항을 완벽하게 충족
- 32K 기본 컨텍스트, YaRN으로 128K까지 확장 가능
- 다중 이미지 및 비디오 입력 지원
- 구조화된 JSON 출력 지원 (송장, 양식, 표 등)
- H100 80GB에서 LoRA/QLoRA 파인튜닝 가능
- 문서 및 차트 이해에 강점 (DocVQA 95.7, ChartQA 87.3)
- Apache 2.0 라이센스로 상업적 사용 가능
- 활발한 커뮤니티 및 문서 지원

**예상 GPU 메모리 사용량:**
- 추론 (bfloat16): ~16GB
- LoRA 파인튜닝: ~30-40GB
- QLoRA 파인튜닝: ~20-25GB

### 2순위: Qwen2.5-VL-3B-Instruct
**선정 이유:**
- 7B 모델과 동일한 기능 제공
- 더 적은 메모리 사용으로 빠른 학습 가능
- 성능은 7B보다 약간 낮지만 충분히 실용적
- 더 큰 배치 사이즈로 학습 가능

**예상 GPU 메모리 사용량:**
- 추론 (bfloat16): ~8GB
- LoRA 파인튜닝: ~20-25GB
- QLoRA 파인튜닝: ~15-20GB

### 3순위: LLaVA-OneVision-Qwen2-7B
**선정 이유:**
- 32K 컨텍스트 지원
- 다중 이미지/비디오 특화
- Qwen2 기반으로 안정적
- 파인튜닝 예제 및 문서 풍부

**단점:**
- Qwen2.5-VL보다 최신 기능 부족
- 구조화된 출력 지원이 Qwen2.5-VL보다 약함

- 컨텍스트 길이 최적화 (토큰 수 관리)
